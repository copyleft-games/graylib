* grl-preview-host --- Live-Reload Sketch Tool
:PROPERTIES:
:CUSTOM_ID: grl-preview-host--live-reload-sketch-tool
:END:
=grl-preview-host= is an optional developer tool that watches a C "sketch" source file, recompiles it on every save, and hot-reloads it into a running window without restarting. On compile failure the last successfully loaded sketch stays active.

*Platform:* Linux only (inotify + dlopen). macOS and Windows are future work.

*This tool is not built by default.* Enable it with =BUILD_PREVIEW=1=.

--------------

** Building
:PROPERTIES:
:CUSTOM_ID: building
:END:
#+begin_src sh
# From the graylib repo root:
make BUILD_PREVIEW=1 preview-tools

# Or directly:
cd tools/preview && make
#+end_src

The resulting binary is at =build/preview/grl-preview-host=.

--------------

** Running
:PROPERTIES:
:CUSTOM_ID: running
:END:
Because the host dynamically links =libgraylib.so=, the library must be on the dynamic linker path at runtime:

#+begin_src sh
LD_LIBRARY_PATH=build/lib:$LD_LIBRARY_PATH \
    ./build/preview/grl-preview-host my-sketch.c
#+end_src

You can also set =GRAYLIB_ROOT= to override the automatic repo-root detection used when compiling sketches at runtime:

#+begin_src sh
GRAYLIB_ROOT=/path/to/graylib \
    LD_LIBRARY_PATH=/path/to/graylib/build/lib:$LD_LIBRARY_PATH \
    grl-preview-host my-sketch.c
#+end_src

*** Export mode (headless GIF)
:PROPERTIES:
:CUSTOM_ID: export-mode-headless-gif
:END:
#+begin_src sh
grl-preview-host my-sketch.c --export out.gif [--frames N] [--fps F]
#+end_src

In export mode no window is opened. The host renders =N= frames at =F= fps (defaults: 120 frames, 60 fps) and writes them to an animated GIF using =GrlGifWriter=. This path requires no display and is useful in CI or for sharing animations.

--------------

** Workflow
:PROPERTIES:
:CUSTOM_ID: workflow
:END:
1. Copy =tools/preview/sketch-template.c= to a new file, e.g. =my-sketch.c=.
2. Start the host:
   #+begin_src sh
   LD_LIBRARY_PATH=build/lib:$LD_LIBRARY_PATH \
       build/preview/grl-preview-host my-sketch.c
   #+end_src

3. Edit =my-sketch.c= in your editor. On every save the host detects the write (via inotify =IN_CLOSE_WRITE= or =IN_MOVED_TO=), waits 150 ms for the write to settle (debounce), then recompiles and reloads.
4. Compile errors are printed to stderr; the previous sketch keeps running.

--------------

** Sketch ABI
:PROPERTIES:
:CUSTOM_ID: sketch-abi
:END:
A sketch is a C source file compiled by the host as a shared object (=-shared -fPIC=). It must *not* define a =main()= function.

Include the graylib umbrella header:

#+begin_src C
#define GRAYLIB_INSIDE
#define GRAYLIB_COMPILATION
#include <graylib.h>
#+end_src

*** Required symbol
:PROPERTIES:
:CUSTOM_ID: required-symbol
:END:
#+begin_src C
void sketch_draw (GrlImage *canvas, double t);
#+end_src

Called every frame. =canvas= is the shared RGBA8 =GrlImage= (800 × 600 by default); =t= is elapsed time in seconds since the host started.

Always call =grl_image_clear_background()= at the top unless you intentionally accumulate pixels across frames.

Example:

#+begin_src C
void
sketch_draw (GrlImage *canvas, double t)
{
    g_autoptr (GrlColor) bg = grl_color_new (20, 18, 30, 255);
    grl_image_clear_background (canvas, bg);

    int cx = grl_image_get_width  (canvas) / 2;
    int cy = grl_image_get_height (canvas) / 2;
    int r  = (int)(40.0 + 20.0 * sin (t * 2.0));

    g_autoptr (GrlColor) col = grl_color_new (220, 80, 255, 255);
    grl_image_draw_circle (canvas, cx, cy, r, col);
}
#+end_src

*** Optional symbols
:PROPERTIES:
:CUSTOM_ID: optional-symbols
:END:
All three optional symbols are looked up with =dlsym=. They may be present individually or in any combination.

**** =void sketch_init (void)=
:PROPERTIES:
:CUSTOM_ID: void-sketch_init-void
:END:
Called once immediately after the first successful load. Use it to allocate heap resources that persist across frames.

**** =void *sketch_state (void)=
:PROPERTIES:
:CUSTOM_ID: void-sketch_state-void
:END:
Called by the host just before closing the current =.so= handle on a hot-reload. Return a pointer to any state you want to survive the reload (e.g. a heap-allocated struct). Return =NULL= if you have nothing to preserve.

The host does not inspect or free this pointer; it is handed back to =sketch_reload()= verbatim.

**** =void sketch_reload (void *state)=
:PROPERTIES:
:CUSTOM_ID: void-sketch_reload-void-state
:END:
Called instead of =sketch_init()= when a hot-reload succeeds and =state= is non-NULL. =state= is the value returned by =sketch_state()= from the previous version. Restore your fields from it and free the old allocation when you no longer need it.

*** State preservation contract
:PROPERTIES:
:CUSTOM_ID: state-preservation-contract
:END:
| =sketch_state= present? | returns non-NULL? | =sketch_reload= present? | Host action after reload                          |
|-------------------------+-------------------+--------------------------+---------------------------------------------------|
| No                      | ---               | ---                      | calls =sketch_init()= if present                  |
| Yes                     | No (NULL)         | ---                      | calls =sketch_init()= if present                  |
| Yes                     | Yes (ptr)         | Yes                      | calls =sketch_reload(ptr)=                        |
| Yes                     | Yes (ptr)         | No                       | calls =sketch_init()= if present; =ptr= is leaked |

*** Minimal sketch (no state)
:PROPERTIES:
:CUSTOM_ID: minimal-sketch-no-state
:END:
#+begin_src C
#define GRAYLIB_INSIDE
#define GRAYLIB_COMPILATION
#include <graylib.h>
#include <math.h>

void
sketch_draw (GrlImage *canvas, double t)
{
    g_autoptr (GrlColor) bg  = grl_color_new (10, 10, 20, 255);
    g_autoptr (GrlColor) col = grl_color_new (80, 180, 255, 255);

    int cx = grl_image_get_width  (canvas) / 2;
    int cy = grl_image_get_height (canvas) / 2;

    grl_image_clear_background (canvas, bg);
    grl_image_draw_circle (canvas, cx, cy, (int)(30.0 + 20.0 * sin (t)), col);
}
#+end_src

** Keep-last-good behaviour
:PROPERTIES:
:CUSTOM_ID: keep-last-good-behaviour
:END:
If recompilation fails (non-zero =cc= exit status) or =dlopen= / =dlsym= fails, the host:

- Prints the compiler output to stderr.
- Leaves the currently loaded =.so= handle in place.
- Continues rendering with the previous sketch.
- Watches for the next file-write event and tries again.

The first load is special: if it fails the host starts with a visible red "No valid sketch loaded" placeholder and waits for a successful build.

In export mode (=--export=), a failed initial compile is a hard error and the host exits with a non-zero status.

--------------

** In-process crash limitation
:PROPERTIES:
:CUSTOM_ID: in-process-crash-limitation
:END:
The sketch shared object is loaded into the *same process* as the host. A =SIGSEGV=, =SIGABRT=, stack overflow, or any other fatal signal originating in sketch code will crash the entire host.

Process isolation (fork/exec + shared-memory canvas) is planned as future work. In the meantime, save frequently and use a debug build (=DEBUG=1 make BUILD_PREVIEW=1 preview-tools=) to get stack traces with gdb.

--------------

** How the host compiles sketches
:PROPERTIES:
:CUSTOM_ID: how-the-host-compiles-sketches
:END:
At runtime the host invokes =cc= like this (simplified):

#+begin_example
cc -shared -fPIC -O2
   -DGRAYLIB_INSIDE -DGRAYLIB_COMPILATION
   $(pkg-config --cflags glib-2.0 gobject-2.0 gio-2.0)
   -I<graylib-root> -I<graylib-root>/src
   <sketch.c>
   -o /tmp/grl-preview-<pid>.so
   -L<graylib-root>/build/lib -lgraylib
   $(pkg-config --libs glib-2.0 gobject-2.0 gio-2.0)
   <graylib-root>/deps/raylib/src/libraylib.a
   -lGL -lm -lpthread -ldl -lrt -lX11
   -Wl,-rpath,<graylib-root>/build/lib
#+end_example

=<graylib-root>= is resolved automatically by walking up from =/proc/self/exe= looking for =config.mk=, or from the =GRAYLIB_ROOT= environment variable.

The compiled =.so= is written to =/tmp/grl-preview-<pid>.so= and deleted on exit.
